# Unit Test Manager Skill

이 스킬은 프로젝트의 코드 품질을 유지하기 위해 유닛 테스트를 작성하고, 이를 `specs/unit_test.md` 문서에 실시간으로 반영하여 테스트 커버리지와 명세를 관리하는 것을 목표로 합니다.

## 핵심 원칙 (Core Principles)

1. **Co-location 전략 (Proximity)**
   - 테스트 코드는 테스트 대상 파일(`*.js` 혹은 `*.logic.js`) 바로 옆에 `[파일명].test.js` 형식으로 배치합니다.
   - 로직과 테스트를 물리적으로 가까이 두어 동기화가 깨지지 않도록 합니다.

2. **철저한 명세 업데이트 (Specification First)**
   - 새로운 유닛 테스트(Unit Test)를 생성하거나 기존 테스트를 대폭 수정할 때마다 반드시 `specs/unit_test.md` 파일을 업데이트합니다.
   - 테스트 파일명, 검증 목표, 주요 테스트 케이스 요약을 문서에 기록합니다.

3. **Jest 표준 준수 (Standardization)**
   - 테스트 도구로 `Jest`를 사용하며, `describe`와 `it`/`test` 블록을 활용하여 테스트 시나리오를 읽기 쉽게 구조화합니다.
   - 엣지 케이스(Null, Empty, Error)에 대한 검증을 반드시 포함합니다.

4. **단위 테스트 실행 및 확인**
   - 테스트 작성 후에는 실제로 `npx jest [파일명]` 명령어를 실행하여 성공 여부를 확인합니다.
   - 실패하는 테스트를 문칭하지(Pass 시키지) 않습니다.

## unit_test.md 기록 가이드

```markdown
### [테스트 파일명] ([경로])
- **검증 목표**: 이 테스트가 보호하고자 하는 핵심 로직 설명.
- **주요 테스트 케이스**:
  - 케이스 1: 정상적인 데이터 입력 시 기대 결과.
  - 케이스 2: 엣지 케이스(예: 데이터 누락) 처리 확인.
  - 케이스 3: 비즈니스 규칙(예: 한도 초과) 동작 확인.
```

## 사용 방법

사용자가 "유닛 테스트 만들어줘" 또는 "로직 검증해봐"라고 요청하면, 작업을 수행한 후 이 스킬의 지침에 따라 `*.test.js` 파일을 작성하고 `unit_test.md`를 업데이트합니다.
