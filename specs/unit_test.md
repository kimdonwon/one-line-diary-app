# 🧪 Unit Test Specification (unit_test.md)

이 문서는 프로젝트의 개별 기능(함수, 상수, 유틸리티)을 검증하기 위한 유닛 테스트 명세를 관리합니다.

---

## 1. 테스트 환경

- **도구**: `Jest`
- **전략 (Co-location)**:
  - 테스트 대상 파일과 동일한 위치에 `[파일명].test.js` 파일을 배치합니다.
  - 별도의 테스트 폴더를 찾아 헤매지 않고 로직 수정 시 즉시 테스트를 업데이트할 수 있도록 합니다.

---

## 2. 테스트 실행 방법

터미널에서 아래 명령어를 입력하여 모든 유닛 테스트를 실행할 수 있습니다.

```bash
npx jest
```

특정 파일만 테스트하고 싶은 경우:

```bash
npx jest src/utils/wordAnalyzer.test.js
```

---

## 3. 테스트 명세 현황

### 3.1 시스템 한도 검증 (`src/constants/limits.test.js`)

앱 전체의 정책을 결정하는 상수값이 설계대로 유지되는지 검증합니다.

- **기본 시스템 한도**: 텍스트 박스 최대 글자 수(`MAX_TEXT_LENGTH`), 일기당 최대 생성 페이지 수(`MAX_PAGES`) 확인.
- **무료 등급 제한 (Free Tier)**: 무료 버전 사용자의 최대 스티커 수(`MAX_STICKERS`), 텍스트 상자 수(`MAX_TEXTS`), 사진 첨부 수(`MAX_PHOTOS`) 확인.
- **프리미엄 등급 제한 (Premium Tier)**: 프리미엄 버전 사용자의 최대 스티커 수, 텍스트 상자 수, 사진 첨부 수 확인.
- **등급별 혜택 비교**: 프리미엄 버전이 무료 버전보다 모든 항목(스티커, 텍스트 상자, 사진)에서 더 높은 한도를 제공하는지 비교 검증.

### 3.2 단어 분석 엔진 검증 (`src/utils/wordAnalyzer.test.js`)

일기 데이터를 분석하여 키워드를 뽑아내는 핵심 알고리즘을 검증합니다.

- **텍스트 정제**: 특수문자, 숫자, 이모지를 제외하고 순수 텍스트만 추출하는지 확인.
- **불용어 필터링**: 한국어 조사('은', '는', '이', '가') 및 무의미한 1글자 단어가 결과에서 제거되는지 확인.
- **멀티 소스 수집**: JSON 형태의 멀티페이지 본문과 캔버스 위 텍스트 상자(`DraggableText`)를 모두 통합해서 파싱하는지 확인.
- **빈도 합산**: 여러 개의 일기 데이터가 들어왔을 때 단어 빈도를 정확히 합산하고 상위 N개를 내림차순으로 정렬하는지 검증.

### 3.3 데이터 백업 및 복구 모듈 검증 (`src/utils/backupRestore.test.js`)

- **검증 목표**: 기기 자체 백업 생성(ZIP 패키징)과 암호화, 그리고 파일 복구 프로세스의 무결성 검증.
- **주요 테스트 케이스**:
  - **백업 생성 시**: SQLite DB로부터 데이터를 잘 추출하는지 검증.
  - **암호화 적용**: 추출된 일기 데이터가 고정 공유 키를 통해 AES 암호화되어 반환되는지 확인 (Mocking으로 대체 검증).
  - **복구 흐름**: 올바른 백업 파일을 선택했을 때 압축을 풀고 data.json을 정상 복호화하는지 확인.

### 3.4 일기 작성 로직 검증 (`src/screens/WriteScreen/WriteScreen.logic.test.js`)

일기 작성 화면의 핵심 비즈니스 로직과 시스템 제약 조건이 사용자 등급 및 상태에 따라 정확히 작동하는지 검증합니다.

- **페이지 생성 제한**: 전체 시스템 최대 페이지 수(`MAX_PAGES`)를 초과하여 페이지를 생성할 수 없는지, 그리고 한도 도달 시 경고 알림(`Alert`)이 발생하는지 확인.
- **무료 등급(Free Tier) 자산 제한**: 무료 사용자가 스티커 및 텍스트 상자를 정해진 한도까지만 추가할 수 있는지, 한도 초과 시 제한 안내 모달이 표시되는지 검증.
- **프리미엄 등급(Premium Tier) 자산 제한**: 프리미엄 혜택이 활성화된 사용자가 더 높은 한도의 자산을 정상적으로 사용할 수 있는지 확인.
- **광고 보상(Ad Reward) 연동**: 무료 사용자가 광고 시청 보상을 받았을 때, 현재 적용된 스티커 부착 한도가 일시적으로 확장되어 추가 부착이 가능한지 검증.

### 3.5 Draggable Text 통합 테스트 (`src/components/DraggableText/DraggableText.test.js`)

'한글 입력 튐 버그'를 해결하기 위해 적용된 비제어 입력 구조(Uncontrolled Input)와 동적 레이아웃 크기 고정 로직 등 `DraggableText`의 통합 생명주기를 방어 기제로 검증합니다.

- **케이스 1 (초기화)**: 초기 렌더링 시 외부 `text` 프롭이 `defaultValue`로 주입되며, `placeholder`와 철저히 분리되어 동작하는지 확인합니다.
- **케이스 2 (스타일 고정)**: 편집 모드(`isEditing`) 진입 시, 레이아웃 흔들림 방지를 위해 내부 `View`와 `TextInput`의 너비 속성이 `100%`로 강제 고정되는지 검증합니다.
- **케이스 3 (편집 완료)**: 텍스트 입력 후 Blur(편집 종료) 시 100ms 지연 타이머가 작동한 후, `onTextChange` 콜백이 최종 수정된 데이터와 함께 정확히 호출되는지 확인합니다.
- **케이스 4 (자동 삭제/엣지 케이스)**: 사용자가 기존 텍스트를 모두 지우고 공백(`'   '`)만 남긴 채 편집을 종료하면, 수정이 아닌 `onDelete` 콜백을 호출하여 객체가 스스로 제거되는지 검증합니다.
- **케이스 5 (외부 동기화)**: 편집 모드가 아닐 때 부모로부터 전달받은 신규 `text` 프롭에 대해, 컴포넌트 리마운트(Re-mount) 없이 `setNativeProps`를 통해 값만 안전하게 동기화되는지 확인합니다.
