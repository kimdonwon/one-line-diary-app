# 스크랩북 캔버스 개선 사양 (v2)

이 문서는 일기 작성 화면(WriteScreen)의 디지털 스크랩북 캔버스에 대한 개선 사항을 기록합니다.

**참고 스킬**: Modular UI Developer, Logic Documenter, Notion Style Designer
**참고 도구**: Context7 (React Native PanResponder 문서)

---

## 1. 롱프레스 기반 드래그 시스템

### 1.1 동작 원리
- **모든 드래거블(스티커, 사진, 텍스트)은 400ms 이상 꾹 눌러야 드래그가 활성화됩니다.**
- 기존의 즉시 드래그 방식을 폐기하고 `useDraggable.js`의 `onPanResponderGrant`에서 `longPressTimer`를 시작.
- 400ms 내에 10px 이상 움직이면 롱프레스 판정 취소 (스크롤 의도로 간주).

### 1.2 선택 상태 진입
- **꾹 누르기를 떼면(Release)** 자동으로 **선택 상태(isSelected = true)**가 됩니다.
- 선택 상태에서는 **회전 핸들**과 **크기 조절**이 가능합니다.
- 텍스트의 경우 선택 상태에서 **수정 버튼**도 표시됩니다.
- 선택 상태는 **5초 후 자동 해제**됩니다 (기존 3초 → 5초로 확대).

### 1.3 선택 해제
- **빈 화면을 탭하거나 다른 드래거블을 탭하면** 기존 선택이 풀립니다.
- 캔버스의 `Pressable.onLongPress`가 새 텍스트를 생성할 때 기존 선택은 자연스럽게 해제됩니다.

### 1.4 인스타그램 스타일 플로팅 쓰레기통
- **드래그 중에만** 캔버스 중앙 하단에 원형의 다크 반투명 쓰레기통이 나타납니다.
- 더블탭 삭제 기능을 **완전히 제거**하고, 대신 드래그 앤 드롭 방식으로 삭제합니다.

#### 시각적 피드백 (Instagram Style)
| 상태 | 배경 | 형태/크기 | 아이콘 | 설명 |
|------|------|---------|--------|--------|
| 일반 | `rgba(0,0,0,0.45)` | 64px 원형 | 24px 화이트 | 텍스트 없이 깔끔한 아이콘만 표시 |
| 활성(위에 올림) | `rgba(235,87,87,0.95)` | `scale: 1.25` | 24px 화이트 | 크기가 확 커지며 강렬한 빨간색 피드백 |

#### 기술적 사양
- **형태**: 완벽한 원형 (Width/Height: 64px, BorderRadius: 32px)
- **그림자**: 깔끔한 드롭 섀도우 (shadowOpacity: 0.2, shadowRadius: 8)
- **애니메이션**: 활성 상태 시 1.25배로 크기가 역동적으로 펌핑(Pulse).
- **텍스트 제거**: 시각적 방해를 줄이기 위해 안내 문구는 완전 제거 처리(`display: 'none'`).

#### 삭제 흐름
```
[선택 상태] 또는 [롱프레스 완료] → [드래그 시작] → [isDraggingAny=true, 다크 원형 쓰레기통 표시]
→ [하단 영역 진입: isOverTrash=true, 쓰레기통이 빨갛게 변하며 확대]
→ [손 떼기: handleDragDrop → 타입에 맞는 삭제 함수 호출]
```

#### 타입별 삭제 래퍼
| 구분 | DragMove 콜백 | DragDrop 콜백 |
|------|--------------|--------------|
| 스티커 | `handleStickerDragMove` | `handleStickerDragDrop` |
| 사진 | `handlePhotoDragMove` | `handlePhotoDragDrop` |
| 텍스트 | `handleTextDragMove` | `handleTextDragDrop` |

## 2. 텍스트 수정 기능

### 2.1 수정 버튼
- 텍스트가 **선택 상태일 때** 좌측 하단에 ✏️ **수정 버튼**이 표시됩니다.
- 우측 하단에는 기존 **회전 핸들**이 위치하며 대칭 구도를 이룹니다.
- 수정 버튼의 디자인은 회전 핸들과 동일한 36px 원형, 흰색 배경, 크라프트 테두리.

### 2.2 수정 모드 진입
- 수정 버튼을 누르면 텍스트가 **편집 모드(isEditing=true)**로 전환됩니다.
- `TextInput`이 활성화되고 자동으로 포커스를 받습니다.
- **기존**: 짧은 탭 2번(선택→편집) → **변경**: 선택 상태에서 수정 버튼만 가능.

### 2.3- 수정 종료: 빈 화면이나 다른 요소를 탭하면 수정 상태와 선택이 동시에 해제됩니다.
- 글자수 제한: **한 텍스트 박스 당 최대 200자**까지 입력 가능합니다. (디자인 유지 및 성능 고려)
- 해제 시 텍스트는 자동으로 저장됩니다 (`onTextChange` 호출).
- 빈 텍스트가 되면 자동 삭제됩니다 (`onDelete` 호출).

## 3. 텍스트 프리셋 패널 변경

### 3.1 제거된 요소
- ❌ 문구 입력 TextInput
- ❌ "추가" 버튼

### 3.2 유지/추가된 요소
- ✅ 폰트 프리셋 선택 (단정 고딕, 감성 명조, 삐뚤빼뚤, 도트체)
- ✅ 텍스트 색상 선택 (9가지)
- ✅ 하이라이터 색상 선택 (5가지)
- ✅ 안내 텍스트: "✏️ 빈 공간을 꾹 눌러서 글을 써보세요"

### 3.3 프리셋 적용 방식
- 패널에서 폰트/색상을 설정하면 **다음에 생성되는 텍스트에 자동 적용**됩니다.
- 상태: `nextTextFont`, `nextTextColor`, `nextTextBgColor` (WriteScreen.logic.js에서 관리)
- 캔버스를 꾹 눌러 새 텍스트를 생성할 때 위 프리셋이 적용됩니다.

## 5. 레이아웃 안정성 규칙 (Layout Stability Rule)

드래거블 요소가 선택(Selection) 상태로 전환될 때 테두리(Border)가 추가되면서 레이아웃이 미세하게 튀는 현상(Layout Shift)을 방지하기 위한 기술적 규칙입니다.

- **항상 테두리 영역 확보**: 모든 드래거블 컨테이너는 비선택 상태에서도 `borderWidth`를 선택 상태와 동일하게 유지하고 `borderColor: 'transparent'`를 적용합니다.
- **적용 이유**: React Native의 Box Model에서 `borderWidth`는 요소의 전체 크기에 영향을 주므로, 동적으로 추가/제거될 경우 내용물이 밀려나는 현상이 발생합니다.
- **구현 예시 (`DraggableText.styles.js`)**:
  ```javascript
  container: {
      borderWidth: 1.5,
      borderColor: 'transparent',
  },
  selected: {
      borderColor: 'rgba(...)',
      borderStyle: 'dashed',
  }
  ```
- **구현 예시 (`WriteScreen.logic.js` - 텍스트 생성 시 200자 제한)**:
  ```javascript
    const newText = {
        id: Date.now().toString() + Math.random().toString(36).substring(7),
        text: textValue.substring(0, 200), // 최대 200자 제한
        fontId,
        color,
        bgColor,
        x: spawnX,
        y: spawnY,
        rotation: 0,
    };
  ```

## 5. 중앙 집중식 선택 관리 시스템

사용자 경험의 일관성을 위해 모든 드래거블 요소의 선택 상태를 부모에서 관리합니다.

- **전역 상태**: `WriteScreen.logic.js`의 `selectedItemId`가 현재 선택된 단 하나의 아이템 ID를 보관.
- **선택 흐름**: 아이템 롱프레스 해제 → `onSelect(id)` 호출 → 부모 상태 업데이트 → 모든 아이템에 `isSelected` prop 전달.
- **자동 해제**: 캔버스 빈 공간 탭 시 `handleClearSelection` 호출하여 전역 초기화.

## 6. 레이아웃 안정성 및 절대 크기 버튼 (Premium UX)

### 6.1 레이아웃 안정성 규칙 (Layout Stability)
- 선택 시 테두리(`borderWidth`)가 생길 때 상자가 들썩이는 현상을 방지.
- **항상 테두리 영역 확보**: 모든 컨테이너는 기본적으로 `borderWidth: 1.5`와 `borderColor: 'transparent'`를 가짐.

### 6.2 절대 크기 핸들 (Inverse Scale Pattern)
- 물체를 크게 키우면 버튼도 거대해지고, 작게 줄이면 버튼이 사라지는 현상을 방지하기 위한 정밀 보정 기술.
- **동작 원리**: 부모의 `scale` 값을 `interpolate` 하여 버튼의 `scale`과 `offset`을 반비례(Inverse)하게 조정.
- **결과**: 물체 본체의 크기와 상관없이 **회전 핸들과 수정 버튼은 항상 화면상 43px 크기를 유지**함.
